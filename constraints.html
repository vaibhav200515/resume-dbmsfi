SQL Constraints

üìå Introduction

SQL Constraints are rules applied to columns in a table to ensure the accuracy, reliability, and integrity of data.

They restrict the type of data that can be inserted into a column and help enforce business rules at the database level.

Constraints can be applied during table creation or altered afterward.

üìò Common SQL Constraints

  • NOT NULL ‚Äì Ensures that a column cannot have a NULL value.
  • UNIQUE ‚Äì Ensures all values in a column are unique.
  • PRIMARY KEY ‚Äì Uniquely identifies each record and combines NOT NULL + UNIQUE.
  • FOREIGN KEY ‚Äì Enforces a link between records in two tables.
  • CHECK ‚Äì Ensures values in a column meet a specific condition.
  • DEFAULT ‚Äì Sets a default value for a column if none is provided.

üßæ Example: Create a table with constraints

CREATE TABLE students (
  student_id INT PRIMARY KEY,
  name VARCHAR(100) NOT NULL,
  age INT CHECK (age >= 16),
  email VARCHAR(100) UNIQUE,
  course VARCHAR(50) DEFAULT 'Undeclared'
);

‚úÖ Explanation

Constraint Description
PRIMARY KEYEnsures each student_id is unique and not null
NOT NULLname cannot be left empty
CHECK (age >= 16)Only allows students aged 16 or older
UNIQUEemail must be unique across all records
DEFAULTIf no course is provided, "Undeclared" will be used
