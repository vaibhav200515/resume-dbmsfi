SQL Comments

üìå Introduction

Comments in SQL are used to explain and clarify your queries or code. They are ignored by the SQL engine and do not affect query execution.

They are helpful for documentation, debugging, or simply making code easier to understand for others (or yourself later).

SQL supports both single-line and multi-line comments, depending on the database system.

üìò Types of Comments

  • Single-line comment: Starts with --
  • Multi-line comment: Enclosed between /* and */

üßæ Example 1: Single-line Comment

-- This query selects all columns from students table
SELECT * FROM students;

üßæ Example 2: Multi-line Comment

/*
  This query inserts a new student into the table.
  We are providing values for all columns.
*/
INSERT INTO students (student_id, name, age, course, admission_date)
VALUES (105, 'Sneha', 21, 'B.Com', '2024-08-10');

‚úÖ Notes

  • Always use comments to describe the logic behind complex queries.
  • Use multi-line comments when explanations require more detail.
  • Too many unnecessary comments can clutter the code, so keep them meaningful.
