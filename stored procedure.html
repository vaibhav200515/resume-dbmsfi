SQL Stored Procedures

üìå Introduction

A stored procedure is a precompiled collection of one or more SQL statements stored in the database with an assigned name.

Stored procedures help you encapsulate logic like inserting data, validating inputs, updating records, or generating reports. They improve performance, reduce redundancy, and support reusability.

They can also accept input parameters and return results or output values.

üìò Syntax (MySQL Style)

DELIMITER //

CREATE PROCEDURE procedure_name (IN param1 datatype, IN param2 datatype)
BEGIN
   -- SQL statements
END //

DELIMITER ;

üî∏ Use DELIMITER to tell the SQL interpreter where the procedure ends (especially in MySQL).

üßæ Example: Create a Procedure to Insert a Student

DELIMITER //

CREATE PROCEDURE AddStudent (
  IN sid INT,
  IN sname VARCHAR(100),
  IN sage INT,
  IN scourse VARCHAR(50),
  IN sdate DATE
)
BEGIN
  INSERT INTO students (student_id, name, age, course, admission_date)
  VALUES (sid, sname, sage, scourse, sdate);
END //

DELIMITER ;

‚ñ∂Ô∏è Calling the Procedure

CALL AddStudent(104, 'Priya Sharma', 22, 'BBA', '2024-08-01');

‚úÖ Explanation

  • CREATE PROCEDURE defines a new procedure.
  • IN parameters are used to pass data into the procedure.
  • The INSERT statement adds a new student using the passed values.
