PL/SQL Scope and Visibility of a Variable

üìå Introduction

In PL/SQL, the scope of a variable refers to the section of the code where the variable can be accessed or used. The visibility of a variable defines whether that variable can be referenced within a specific part of a block or nested blocks.

Understanding variable scope helps prevent naming conflicts and ensures controlled access to data within different sections of a PL/SQL program.

üìò Key Points

  • Variables declared in an outer block are accessible to all inner blocks.
  • Variables declared in an inner block are not visible to the outer block.
  • PL/SQL supports nested blocks, and each block can have its own set of variables.

üßæ Example

DECLARE
  v_outer NUMBER := 50;
BEGIN
  DECLARE
    v_inner NUMBER := 100;
  BEGIN
    DBMS_OUTPUT.PUT_LINE('Inner Block: v_outer = ' || v_outer);
    DBMS_OUTPUT.PUT_LINE('Inner Block: v_inner = ' || v_inner);
  END;

  DBMS_OUTPUT.PUT_LINE('Outer Block: v_outer = ' || v_outer);
  -- The next line would cause an error because v_inner is not visible here
  -- DBMS_OUTPUT.PUT_LINE('Outer Block: v_inner = ' || v_inner);
END;
