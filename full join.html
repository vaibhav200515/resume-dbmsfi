SQL FULL JOIN

üìå Introduction

FULL JOIN (also known as FULL OUTER JOIN) returns all records when there is a match in either left or right table.

It combines the results of both LEFT JOIN and RIGHT JOIN. Non-matching rows will have NULLs for the missing side.

It is useful when you want a complete view of data from both tables, regardless of matching records.

üìò Syntax

SELECT columns
FROM table1
FULL JOIN table2
ON table1.column_name = table2.column_name;

üßæ Example: FULL JOIN students and courses

SELECT students.name, courses.course_name
FROM students
FULL JOIN courses
ON students.course_id = courses.course_id;

‚úÖ Explanation

This query retrieves all students and all courses. If a student is not enrolled in any course or if a course has no students, NULLs will be shown for the missing data.
