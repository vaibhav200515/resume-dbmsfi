SQL NULL Functions

üìå Introduction

In SQL, NULL represents a missing or undefined value. When performing calculations or displaying data, NULL values can affect results.

To handle such cases, SQL provides NULL functions that help you replace, check, or transform NULL values into meaningful defaults.

These functions are very useful for ensuring clean output in reports or preventing unexpected results in computations.

üìò Common NULL Functions

  • ISNULL(expr, replacement) ‚Äì SQL Server
  • IFNULL(expr, replacement) ‚Äì MySQL
  • COALESCE(expr1, expr2, ...) ‚Äì All SQL dialects
  • NVL(expr, replacement) ‚Äì Oracle

üßæ Example 1: Using ISNULL (SQL Server)

SELECT name, ISNULL(phone, 'Not Provided') AS contact_number
FROM customers;

‚úÖ Replaces NULL in the phone column with 'Not Provided'.

üßæ Example 2: Using IFNULL (MySQL)

SELECT name, IFNULL(email, 'No Email') AS contact_email
FROM employees;

‚úÖ Shows 'No Email' where the email column is NULL.

üßæ Example 3: Using COALESCE (All SQL)

SELECT name, COALESCE(address, city, 'Unknown') AS location
FROM users;

‚úÖ Returns the first non-NULL value among address, city, or 'Unknown'.

üßæ Example 4: Using NVL (Oracle)

SELECT name, NVL(salary, 0) AS salary_amount
FROM employees;

‚úÖ If salary is NULL, it will return 0.
