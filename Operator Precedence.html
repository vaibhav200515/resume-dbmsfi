PL/SQL Operator Precedence

üìå Introduction

In PL/SQL, expressions can include multiple operators. The operator precedence determines the order in which these operators are evaluated.

Understanding operator precedence ensures that complex expressions are calculated as expected, especially when arithmetic, comparison, and logical operators are used together.

üìò Precedence Order (Highest to Lowest)

Operator(s) Type Associativity
**ExponentiationRight to Left
+, - (Unary)SignRight to Left
*, /Multiplication/DivisionLeft to Right
+, -Addition/SubtractionLeft to Right
||ConcatenationLeft to Right
=, !=, <>, <, >, <=, >=ComparisonLeft to Right
IS [NOT] NULL, LIKE, IN, BETWEENLogical comparisonLeft to Right
NOTLogical NOTRight to Left
ANDLogical ANDLeft to Right
ORLogical ORLeft to Right

üßæ Example

DECLARE
  result NUMBER;
BEGIN
  result := 10 + 2 * 3; -- Multiplication happens before addition
  DBMS_OUTPUT.PUT_LINE('Result: ' || result); -- Outputs 16
END;
/

‚úÖ Explanation

Expression Evaluated As
10 + 2 * 310 + (2 * 3) = 16

Multiplication has higher precedence than addition, so 2 * 3 is calculated first.

üí° Tip

Use parentheses ( ) to explicitly define the desired evaluation order, especially when mixing different types of operators. This improves both accuracy and readability.

result := (10 + 2) * 3; -- Now result is 36
