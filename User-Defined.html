User-Defined Exceptions in PL/SQL

üìå Introduction

User-defined exceptions in PL/SQL are custom errors that developers define to handle specific error scenarios that are not covered by predefined exceptions.

They are explicitly declared using the EXCEPTION keyword and raised using the RAISE statement.

This allows greater control over error management, making programs more robust and easier to maintain.

üõ†Ô∏è Steps to Use User-Defined Exceptions

  1. Declare the exception in the DECLARE section.
  2. Raise the exception using the RAISE keyword when the condition is met.
  3. Handle the exception in the EXCEPTION section of the block.

üßæ Example: Raise an exception if salary is below minimum

DECLARE
  v_salary NUMBER := 2000;
  e_low_salary EXCEPTION;
BEGIN
  IF v_salary < 3000 THEN
    RAISE e_low_salary;
  END IF;
  DBMS_OUTPUT.PUT_LINE('Salary is acceptable.');
EXCEPTION
  WHEN e_low_salary THEN
    DBMS_OUTPUT.PUT_LINE('Error: Salary is below the minimum required.');
END;
