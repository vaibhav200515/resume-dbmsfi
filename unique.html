SQL UNIQUE Constraint

üìå Introduction

The UNIQUE constraint in SQL ensures that all values in a column or a combination of columns are different from each other.

It helps maintain data integrity by preventing duplicate entries in the specified column(s).

You can use UNIQUE on one or more columns. Multiple UNIQUE constraints are allowed in a table, but only one PRIMARY KEY.

üìò Syntax

CREATE TABLE table_name (
  column1 datatype UNIQUE,
  column2 datatype,
  ...
);

You can also add UNIQUE after table creation:

ALTER TABLE table_name
ADD CONSTRAINT constraint_name UNIQUE (column_name);

üßæ Example: Enforce UNIQUE constraint on email

CREATE TABLE students (
  student_id INT,
  name VARCHAR(100),
  email VARCHAR(100) UNIQUE
);
